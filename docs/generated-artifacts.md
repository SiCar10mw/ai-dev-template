# Generated Artifacts

The template uses generate-don't-maintain for derived evidence.

## Mandatory Rule

All derived artifacts are generated by `scripts/gen_*` commands, regenerated by CI, and never hand-edited.

This includes:

- architecture diagrams
- executive and committee decks
- DOCX deliverables
- AIBOM
- governance evidence
- M365 publication plans

## Committed Evidence

Committed generated artifacts live under `generated/`. They are evidence, so they are source-controlled and protected by
a drift tripwire.

```bash
python scripts/gen_all_artifacts.py
python scripts/check_generated_artifacts.py
```

## Drift Tripwire

`scripts/check_generated_artifacts.py` regenerates the artifacts into a temporary directory and compares them against the
committed `generated/` directory. The build fails if any generated artifact is missing, unexpected, or byte-stale.

## Principle Tripwire Audit

`scripts/check_principle_tripwires.py` verifies that every mandatory principle in `STANDARDS.md` appears in
`config/mandatory-principles.json`, has a negative probe, and names at least one discoverable build-failing tripwire.

## Local Drafts

Local scratch output goes under `out/`, which is ignored by git. Do not treat `out/` as evidence.
