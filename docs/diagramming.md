# Diagramming

Diagrams must be maintainable source artifacts, not unexplained screenshots.

## Default Tooling

| Use Case | Recommended Default |
|---|---|
| Collaborative architecture diagrams | Lucid Chart via approved MCP or connector |
| Source-controlled technical diagrams | Mermaid, PlantUML, Graphviz, or a code-generated diagram |
| Python-generated static images | `diagrams`, `matplotlib`, or a project-specific renderer |
| Public docs display | exported PNG/SVG in `docs-site/static/` with source reference |

## Rules

- Keep the source of truth clear: Lucid document URL, Mermaid/PlantUML source, or generator script.
- Store exported images in-repo when they are needed for docs or review.
- Update the docs page that embeds a diagram in the same change that updates the diagram.
- Treat Lucid edits as external writes. Create, edit, share, and publish actions require human approval.
- If a diagram is generated by code, include a deterministic smoke test or regeneration command.
- Committed generated diagrams live under `generated/architecture/` and must pass `scripts/check_generated_artifacts.py`.

## Recommended Locations

- `docs/diagrams/` for internal diagram source notes.
- `docs-site/static/img/` for public images.
- `scripts/` for generated diagram scripts.
- `generated/architecture/` for committed generated evidence diagrams.
- `out/` for generated local artifacts that are not committed.
